# Validation Checklist

## 美学风格

- [ ] 是否使用浅色基底。
- [ ] 是否使用柔和强调色。
- [ ] 是否保持轻盈、亲和、内容优先。
- [ ] 是否避免厚重暗色、强渐变和重阴影。

## Design Tokens

- [ ] 是否使用 `design-tokens.json` 的语义 token。
- [ ] 是否避免散落硬编码颜色、圆角、阴影和断点。
- [ ] 主色替换后是否同步 hover、pressed、focus、ripple 和 shadow。
- [ ] 文字、图标和背景是否满足对比度要求。

## Layout Grammar

- [ ] 是否使用可组合布局，而不是固定页面顺序。
- [ ] 是否覆盖 hero、section、grid、CTA、footer 或所需替代模块。
- [ ] 内容容器是否有明确宽度和 padding。
- [ ] 固定导航是否正确占位。

## Component Recipes

- [ ] 是否按组件配方生成 Button、Navigation、Card、Form、Modal 等组件。
- [ ] 每个组件是否包含用途、结构、token、状态、响应式和可访问性。
- [ ] 卡片是否保持 16:9 媒体比例和两行标题策略。
- [ ] 图标按钮是否有可访问名称。

## 状态覆盖

- [ ] 是否包含 hover。
- [ ] 是否包含 active。
- [ ] 是否包含 focus。
- [ ] 是否包含 disabled。
- [ ] 是否包含 loading。
- [ ] 是否包含 error。
- [ ] 是否包含 success。
- [ ] 是否包含 empty。

## 响应式

- [ ] 是否包含 desktop、tablet、mobile。
- [ ] 移动端是否没有横向溢出。
- [ ] 触控目标是否不小于 44px。
- [ ] 字号和 section spacing 是否按断点缩放。
- [ ] 图片比例是否稳定。

## 可访问性

- [ ] 文本对比度是否满足 WCAG AA。
- [ ] focus-visible 是否清晰。
- [ ] 键盘是否可操作。
- [ ] reduced motion 是否可用。
- [ ] 表单是否有 label。
- [ ] 错误提示是否关联字段。
- [ ] 图片是否有 alt 策略。
- [ ] 颜色是否不是唯一信息载体。
- [ ] heading 层级是否合理。

## 内容规则

- [ ] 标题是否短而明确。
- [ ] 元信息是否顺序稳定。
- [ ] CTA 文案是否可执行且不过长。
- [ ] 空状态是否温和并给出下一步。
- [ ] 示例文案是否中性、可替换、非专有。

## 发布态清洁度

- [ ] 是否没有特定品牌名称。
- [ ] 是否没有特定域名。
- [ ] 是否没有特定页面名称。
- [ ] 是否没有专有图片、专有图标或专有文案。
- [ ] 是否没有真实业务数据。
- [ ] 是否没有工作过程说明。
- [ ] 是否没有非授权 skill、非授权设计系统或非授权文档名称。
- [ ] 是否能够在没有额外上下文时独立使用。

## AI Agent 可执行性

- [ ] 是否说明输入参数。
- [ ] 是否说明输出模式。
- [ ] 是否说明读取顺序。
- [ ] 是否提供 runtime decision tree。
- [ ] 是否提供 prompt templates。
- [ ] 是否提供 examples。
- [ ] 是否可维护、可扩展、可替换。
